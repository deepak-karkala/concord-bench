## CLI Surfaces to Verify
- `concord run --model <id> --scenario <id>` — runs a single episode
- `concord run-batch --models <ids> --scenarios <split>` — runs a batch
- `concord generate --domain <d> --culture <c> --count N` — synthetic gen via AWM
- `concord grade --episode-log <path>` — re-grades a saved episode
- `concord export --format hf` — exports to Hugging Face dataset format

## Key Interactions to Verify

### Synthetic generation pipeline
- AWM → enrichment → Concord scenario YAML — full happy path
- Cultural adaptation: same scenario across 5 cultures, audit log written
- Repeated-game variants: 5-round sequences emit correct inter-episode state

### Negotiation environment
- Each action type (message, offer, accept, reject, walk_away, escalate)
- Offer extraction via constrained generation (when supported)
- Offer extraction regex fallback (when not supported)
- Max-turns boundary (10 → terminal with no deal)
- Repeated-game state load from prior round

### Agent adapters
- Closed-API: OpenAI, Anthropic, Google — all retry on 429, timeout cleanly
- Open-weight: load Qwen-3, extract activations, apply steering vector

### Graders
- Each code-based grader on its 10 reference scenarios
- Each LLM judge on 30+ calibration transcripts (kappa >= 0.7 with author labels)

## Edge Cases

- Agent emits malformed JSON for offer (must fall back to regex parser)
- Agent never emits an offer in 10 turns (terminal, no deal)
- Agent emits offer that violates hard constraint (logged, episode terminates)
- Agent reveals BATNA in plain text (privacy grader detects)
- Cross-cultural scenario where US-style direct refusal would be culturally inappropriate (cultural_sensitivity grader penalizes)
- Repeated-game round 5 with no round 1-4 history (load failure, fail loudly)
- Open-weight model OOM during activation extraction (catch, retry smaller batch)

## Critical Paths

1. **End-to-end episode:** scripted-buyer vs scripted-seller, asserts deal extracted, all graders run, model card produced.
2. **Frontier smoke:** 1 scenario × Opus 4.7 buyer × scripted seller, asserts no errors, episode logged.
3. **Reproducibility:** same scenario + seed + model snapshot + temperature → identical transcript.
4. **Schema regression:** every scenario YAML in `concord/scenarios/` validates against pydantic schema.
5. **Calibration regression:** LLM judges on the held-out 10 calibration transcripts hit kappa >= 0.7.
